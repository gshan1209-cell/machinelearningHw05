# AI Agent 協作指南 (Agent Collaboration Guide)

這份文件旨在指引所有的 AI Agent (包括但不限於 ChatGPT, Claude, Gemini) 在本專案中如何進行協作與開發。

## 1. 核心開發原則
- **遵守現有架構**：本專案使用 Next.js (前端) + FastAPI (後端)。請勿隨意引入未經許可的大型框架或破壞現有架構。
- **讀取上下文**：在進行任何開發前，請務必先閱讀 `DEVELOPMENT_RECORD.md` 了解當前進度與上一個 Agent 留下的交接事項。
- **維持代碼風格**：前端全面使用 Tailwind CSS 進行現代化樣式排版，避免寫傳統的純 CSS 樣式，並遵守 TypeScript 的型別定義。後端遵守 PEP 8 與 FastAPI 慣例。
- **漸進式修改**：以最小影響範圍完成任務，若必須重構或修改多個檔案，請先在思考過程中列出影響範圍。

## 2. 任務執行流程
1. **讀取文件**：優先讀取 `DEVELOPMENT_RECORD.md` 以及 `Gemini_機器學習前十大演算法互動學習網站_開發規格書.md`。
2. **理解任務**：從使用者提示 (Prompt) 或交接事項中確認當前目標。
3. **執行修改**：提供清晰、無誤的程式碼差異 (diff 格式) 或完整程式碼。
4. **更新紀錄**：任務完成後，務必提示使用者或自行生成更新 `DEVELOPMENT_RECORD.md` 的內容，將已完成事項打勾 ✅，並為下一個 Agent 留下待辦事項 ❌。

## 3. 與使用者的溝通
- 用親切、專業的繁體中文回應。
- 說明程式碼修改原因要簡潔有力，不要給出過度冗長的無用資訊。
- 如果遇到矛盾的需求或規格中未定義的模糊地帶，主動提出選項讓使用者決定。
- 每次交付代碼後，提供簡短的驗證方式或下一步建議。
